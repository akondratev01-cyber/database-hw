# Задание по дисциплине «База данных»

Репозиторий содержит итоговое домашнее задание по дисциплине «База данных».

В рамках работы реализованы 4 базы данных и подготовлены SQL-решения для 13 задач.

## Содержание работы

В работе реализованы 4 базы данных:

1. Транспортные средства
2. Автомобильные гонки
3. Бронирование отелей
4. Структура организации

Каждая база данных включает:
- создание таблиц;
- заполнение тестовыми данными;
- решение SQL-задач разной сложности.

В работе использованы связи между таблицами с помощью внешних ключей (`FOREIGN KEY ... REFERENCES`), что обеспечивает целостность данных между связанными сущностями.

## Структура репозитория

- `01_transport/01_transport_MySQL.sql` — база данных «Транспортные средства»
- `02_racing/02_racing_MySQL.sql` — база данных «Автомобильные гонки»
- `03_hotels/03_hotels_MySQL.sql` — база данных «Бронирование отелей»
- `04_organization/04_organization_MySQL.sql` — база данных «Структура организации»

## Используемая СУБД

- MySQL / MariaDB
- Среда выполнения: DBeaver

## Инструкция по запуску

1. Создать одну из баз данных: `transport_db`, `racing_db`, `hotels_db`, `organization_db`.
2. Открыть нужный SQL-файл в DBeaver.
3. Выполнить команду `USE <имя_базы>;`
4. Запустить скрипт целиком:
    - создание таблиц;
    - заполнение данными;
    - выполнение запросов по задачам.

## Примечание

Все решения оформлены в виде SQL-скриптов с комментариями для удобства проверки.

Для задач с `WITH RECURSIVE` рекомендуется выполнять запрос целиком как один SQL-блок. В DBeaver это удобно делать, установив курсор внутри полного запроса или выделив весь блок до `;`.

При выполнении задачи 5 было обнаружено расхождение между текстом условия и примером ожидаемого вывода.

Для совпадения с примером вывода требуется более мягкий порог при подсчете `low_position_count`, однако в тексте условия указано "больше 3.0".
Поэтому в итоговом решении был выбран вариант, соответствующий формулировке задания: `AVG(position) > 3.0`.
